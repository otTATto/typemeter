# マークアップに関するルールや考え方

## 全般

- semantic なマークアップを意識する
  - むやみに `<div>` タグを多用しない
- 画像に対して alt 属性は必須
  - 内容を説明すべき場合と空文字にすべき場合がある
    - 例えば背景画像など、ユーザーがスクリーンリーダーを通してフォーカスしない要素は空文字にすべきである
    - 一方で、フォーカスしうる画像については説明されるべきである
- CSS は Tailwind CSS を使う
  - ただし、どうしても表現が難しい場合は Vanilla CSS を書いても構わないが、実装前に必ず承認を得る
  - 4px スケール（Tailwind の spacing scale）を基本とする（例: `p-1`, `w-2`,`h-3`）
  - ただし、やむを得ない場合には `h-[10px]` などの任意の px 指定の書き方を許容する
- 最小のターゲットサイズは 44px とする
  - ボタンやリンクのターゲットサイズは 44px 以上の大きさを確保する
  - 文字サイズとは関係なく当たり判定だけを大きくできる
- アイコンはなるべく Lucide が提供するアイコンセット https://lucide.dev/icons/ を使う
  - また、どうしても SVG を使いたい場合は `<svg>` による直書きは避け、`.svg` ファイルを import する
