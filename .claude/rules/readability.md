# 可読性に関するルールや考え方

## 全般

- 1つの関数の責務は明確にする
  - 長すぎる関数は分割する
- 原則としてアロー関数を使用する（const fn = () => {}）
  - ただし、Vue コンポーネントのファイル名は PascalCase とする
  - 上記以外における function 宣言は、明確な理由がある場合にのみ使用する
- 副作用のある処理は明示する
- 条件文のネストが深くなることは避け、早期 return を歓迎する
- 三項演算子は明らかに二者択一の場合にのみ使用し、ネストは深くしない（三者以上の場合は対応表（オブジェクトや Map）を用意する）
