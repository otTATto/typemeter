# typemeter ドキュメント

## 概要

このディレクトリ（`docs/`）は、typemeter の技術ドキュメントの置き場です。この README はその入口であり、どのドキュメントに何が書かれているかを案内する目次の役割を持ちます。

ただし、以下のような内容は扱っていません。

- ドキュメントの書き方のルール（詳しくは、`.claude/rules/documentation.md` を参照してください）
- アプリケーション自体の紹介やセットアップ手順（詳しくは、リポジトリルートの `README.md` を参照してください）

## ドキュメント一覧

テーマ別のドキュメントは `docs/` 直下に、リリースノートは `docs/releases/` に置いています。

| ファイル                               | 何が書かれているか                                                                               | 読むべき人・状況                                                 |
| -------------------------------------- | ------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| [settings.md](./settings.md)           | アプリケーション設定の項目一覧・保存先（`settings.json`）・処理フロー・追加手順                  | 設定項目を追加・変更する人、設定の保存・同期の仕組みを知りたい人 |
| [app-icon.md](./app-icon.md)           | アプリアイコンの原本の所在・再生成手順・Illustrator 書き出し時の注意                             | アイコンを更新する人、アイコンファイルの由来を知りたい人         |
| [app-lifecycle.md](./app-lifecycle.md) | ウィンドウとプロセスのライフサイクル（× = 常駐、トレイ、二重起動防止、終了時の flush）           | ウィンドウやトレイの挙動を変更する人、常駐の仕様を知りたい人     |
| [releases/](./releases)                | 各バージョンのリリースノート（`.github/RELEASE_TEMPLATE.md` から `bun run release:note` で生成） | リリース作業をする人                                             |

## ドキュメントの追加について

新しいドキュメントを追加する際は、`.claude/rules/documentation.md` のルールに従って書いてください。あわせて、この README の[ドキュメント一覧](#ドキュメント一覧)にも 1 行追加し、目次としての役割が保たれるようにしてください。
