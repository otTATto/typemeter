# アプリアイコン

## 概要

このドキュメントでは、typemeter のアプリアイコンについて書いています。具体的には、以下のようなことについて書いています。

- アイコンの原本（デザインデータ）の所在と、リポジトリ内のアセットとの関係
- アイコンを更新したときの、各プラットフォーム向けアイコンの再生成手順
- Adobe Illustrator から SVG を書き出す際の注意点

ただし、以下のような内容は扱っていません。

- アプリ UI のデザイン（Tailwind CSS やテーマ）について（現在、参照するべきドキュメントはありません）

アイコンのデザインを更新する人、またはリポジトリ内のアイコンファイルの由来を知りたい人が読むことを想定しています。

## アイコンの原本と所在

アイコンの原本は開発者の個人 NAS 上にある Adobe Illustrator データ（`.ai`）であり、リポジトリ内の SVG はそこから書き出した成果物です。リポジトリ内の SVG を直接編集することはありません。

リポジトリ内のアセットは以下のとおりです。

| ファイル            | 用途                                                                        |
| ------------------- | --------------------------------------------------------------------------- |
| `app-icon.svg`      | 各プラットフォーム向けアイコンの生成ソース（`tauri icon` の入力）           |
| `public/icon.svg`   | favicon（`index.html` から参照）                                            |
| `src-tauri/icons/*` | `tauri icon` が生成した各プラットフォーム向けアイコン一式（手動編集しない） |

なお、原本にはダークモード用バリエーション（白いキーキャップ）も存在しますが、これはアプリアイコンではなく将来の UI 内での利用（メイン画面のメーター内側など）を想定したもので、現時点ではリポジトリに取り込んでいません。

## アイコンの再生成手順

アイコンを更新する際は、原本の編集から始めて以下の手順でリポジトリに反映します。

1. NAS 上の `.ai` を編集し、SVG を書き出す（書き出し設定は[次節](#illustrator-書き出し時の注意)を参照）
2. 書き出した SVG でリポジトリルートの `app-icon.svg` を差し替える
3. `bun run tauri icon app-icon.svg` を実行し、`src-tauri/icons/` を一括再生成する
4. あわせて生成されるモバイル用アイコン（`src-tauri/icons/ios/`・`src-tauri/icons/android/`）は、デスクトップ専用アプリのため削除する
5. favicon 用に `app-icon.svg` を `public/icon.svg` へコピーする

反映の確認には注意が必要です。`bun run tauri dev` ではウィンドウやタスクバーのアイコンに反映されません。`bun run tauri build` でビルドしてインストールしたもので確認してください。

## Illustrator 書き出し時の注意

SVG 書き出しオプションの「オブジェクト ID」を「レイヤー名」にすると、日本語のレイヤー名がそのまま SVG の `id` 属性に混入します。日本語や空白を含む `id` は XML パーサーや SVG 最適化ツールとの互換性リスクがあるため、書き出し時は「オブジェクト ID: 最小」を選んでください。

もし `id` が混入した SVG しか用意できない場合は、コミット前に `id` 属性を削除してください。アイコン用途の SVG において `id` は描画に寄与しないため、削除しても表示は変わりません。
